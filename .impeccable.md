# MOCA / Cortex — Design Context

## Design Context

### Users
Mixed audience spanning technical and non-technical users. Researchers, analysts, and knowledge workers use MOCA to ingest documents, explore knowledge graphs, and get AI-powered answers with source attribution. Some users are deeply technical (graph exploration, entity extraction config), others just need fast, reliable search and Q&A. The interface must serve both without dumbing down for experts or intimidating newcomers.

### Brand Personality
**Bold, powerful, futuristic** — MOCA is a next-generation knowledge system. It should feel like operating advanced technology that's still intuitive. The single vibrant accent color against a monochrome palette reinforces this: restrained sophistication with moments of energy. Think mission control meets modern design tool.

### Emotional Goals
- **Confidence & control**: Users should feel mastery over their knowledge base. Clear navigation, predictable interactions, visible system state (processing status, search progress, graph context).
- **Trust & reliability**: Transparent source attribution, accurate results, consistent behavior. The interface communicates stability — nothing feels fragile or experimental.

### Aesthetic Direction
- **Visual tone**: Monochrome foundation with a single bold accent. Glass morphism for depth and layering. Dark mode primary. High contrast, clean typography.
- **References**: Notion/Arc (clean, modern, touches of personality) + Perplexity/ChatGPT (AI-native search/chat with source attribution). The intersection: polished AI tool that feels both powerful and approachable.
- **Anti-references**: Avoid generic SaaS dashboards, cluttered enterprise UIs, or overly playful/casual aesthetics. Never look like a prototype or demo.
- **Theme**: Dark mode is the primary experience. Light mode is supported but dark is the default and should receive primary design attention.

### Design Principles

1. **Accent with intent** — The yellow-green accent (`oklch(0.79 0.18 70.67)`) is the single chromatic color. Use it deliberately for CTAs, active states, and key data points. Every use should draw the eye to something actionable or important. Overuse dilutes its power.

2. **Clarity over density** — Serve mixed audiences by making complex information scannable. Use progressive disclosure: surface the essential, let users drill deeper. Never hide critical state, never overwhelm with everything at once.

3. **Motion as communication** — Framer Motion animations should convey state changes and spatial relationships, not decorate. Staggered entrances guide reading order. Transitions confirm actions. Loading states communicate progress. Remove any animation that doesn't serve a communicative purpose.

4. **Glass and depth** — The glass morphism treatment creates visual hierarchy through layering. Background content bleeds through subtly, reinforcing spatial metaphor. Use blur and transparency to separate content planes, not just as decoration.

5. **Trust through transparency** — Every AI-generated answer shows its sources. Every processing job shows its status. Every search result shows its relevance signals. The system never hides how it arrived at a result.

## Design Tokens Reference

### Color System (OKLch)
| Token | Light | Dark | Usage |
|-------|-------|------|-------|
| `--background` | `oklch(1.0 0 0)` | `oklch(0.1448 0 0)` | Page background |
| `--foreground` | `oklch(0.1448 0 0)` | `oklch(0.9851 0 0)` | Primary text |
| `--accent` | `oklch(0.79 0.18 70.67)` | `oklch(0.79 0.18 70.67)` | CTAs, active states, key data |
| `--card` | `oklch(1.0 0 0)` | `oklch(0.2134 0 0)` | Card surfaces |
| `--muted` | `oklch(0.9702 0 0)` | `oklch(0.2686 0 0)` | Subdued backgrounds |
| `--border` | `oklch(0.9219 0 0)` | `oklch(0.3407 0 0)` | Dividers, card borders |
| `--destructive` | `oklch(0.583 0.239 28.48)` | `oklch(0.702 0.189 22.23)` | Error, delete actions |

### Typography
| Role | Font | Weight | Size |
|------|------|--------|------|

| Code | JetBrains Mono | 400 | 14px |

### Spacing Scale
Standard Tailwind scale: `gap-1` (4px) through `gap-6` (24px). Content padding: `p-4` to `p-6`. Section spacing: `mb-6` to `mb-12`.

### Border Radius
`--radius: 0.5rem` (8px default), `--radius-md: 6px`, `--radius-sm: 4px`

### Animation Defaults
- Entrance: `opacity: 0, y: 20` → `opacity: 1, y: 0` over 300-400ms
- Stagger: 50ms between items
- Hover transitions: 300ms ease
- Spring physics for modals: `type: "spring", damping: 25, stiffness: 300`
