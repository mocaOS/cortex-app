# Apps — In-Instance App Hosting

Static web apps (built from [cortex-app-template](https://github.com/mocaOS/cortex-app-template)) installed from a zip and served by the backend under `/apps/{app_id}/`, sandboxed, with proxied least-privilege API access. Part of the app-ecosystem plan (master doc: `cortex-registry/ECOSYSTEM.md`; builder skills: cortexskills.org/builder). **Gated on `ENABLE_APPS` (default off): every route 404s and the admin UI section hides itself — zero traces when disabled** (same philosophy as x402).

## Package & storage

Package = zip of `app.json` (manifest) + `icon.svg` + `dist/` (static bundle). Storage is filesystem-only under `settings.apps_dir` (default `.agents/apps`, mounted as the `apps_data` volume in prod) — no Neo4j schema. **Upgrades (reinstall of the same id) preserve install.json, config.json, `storage.sqlite`(+wal/shm), and `tasks/`** — wiping platform state on a version bump would erase a sync app's dedup keys/cursor/schedules (live-verify regression, tested):

```
{apps_dir}/{app_id}/
  app.json       manifest, verbatim from the package
  icon.svg
  dist/          static bundle; manifest "entry" (index.html) lives here
  install.json   install record: minted key (encrypted), grants, enabled state
  config.json    admin-configured values (secrets Fernet-encrypted, enc: prefix)
```

## Manifest (app.v1)

Server-side validation in `AppService.validate_manifest()` is the twin of the template's `validate.mjs` — **keep them in sync**. Key fields: `id` (kebab slug, `launch` reserved), `version` (semver), `type` (`"static"` and `"platform"` hostable; `"service"` never hosted), `cortex.keyScope` (`read`|`read_write`), `cortex.endpoints` (proxy allowlist, `/api/`-relative), `cortex.collections` (`"user-selected"`|`"all"`|[ids]), `config[]` (same field semantics as skill config: `type: secret`, `auth_header`), `externalHosts` (CSP connect-src, `${VAR}` resolved from config), `sharing.links`, `capabilities` (platform only). Canonical JSON Schema: `cortex-app-template/schema/app.v1.json`.

## Platform capabilities (`type: "platform"`)

`_SUPPORTED_CAPABILITIES = {http, tasks, storage, llm}` (declaring anything else fails install; tasks/storage/llm take `{}` — stray config rejected). Mutating platform calls (storage writes, task control) additionally require token role admin|editor (`_require_app_writer`) — share-link viewers are read-only.

- **`http`** — `POST /apps/{id}/api/platform/http` with envelope `{method, url, body?, content_type?, headers?}` (methods GET/POST/PUT/PATCH/DELETE/PROPFIND — PROPFIND so app UIs can drive WebDAV folder browsers; the browser parses the multistatus XML itself via DOMParser) (extra headers pass the `_FORBIDDEN_APP_HEADERS` denylist: Authorization/Host/framing stripped, config auth wins on collisions; `Cookie` allowed — app values can't carry server secrets, and scraping needs it, e.g. YouTube consent `SOCS=CAI`). Executes server-side with auth headers built from config vars carrying `auth_header` templates (skill-style; secrets never reach the browser, target needs **no CORS setup**). Templates may reference ANY config var by name (longest-name-first substitution) and wrap parts in `base64(...)` — evaluated after substitution, e.g. `Authorization: Basic base64(NC_USER:NC_APP_PASSWORD)`, so Basic-auth apps take plain credentials instead of user-encoded base64 (field lesson: the manual encode step corrupted credentials immediately). `google_sa_token(VAR, scopes)` mints+caches Google service-account tokens server-side (`resolve_google_sa_tokens`: RS256 JWT via cryptography, token_uri pinned to oauth2.googleapis.com, resolved async in execute_app_http; the expression is shielded from var substitution so the raw key JSON never inlines into a header) — the gdrive-sync no-OAuth model. `ms_graph_token(TENANT_VAR, CLIENT_VAR, SECRET_VAR)` is the Microsoft sibling (`resolve_ms_graph_tokens`: client-credentials grant, endpoint pinned to login.microsoftonline.com with the tenant segment sanitized, cached; pair with Sites.Selected + auth_host graph.microsoft.com) — the sharepoint-sync model. A var may add `auth_host` (literal or `${CONFIG_VAR}` ref) to restrict its header to ONE host — required posture for multi-host apps (e.g. venice + youtube), else every declared host receives every credential; unscoped vars inject everywhere (fine single-host). Targets restricted to `capabilities.http.hosts` (literals or `${CONFIG_VAR}` refs resolved from app config → `allowed_http_hosts()`), then SSRF-guarded via `ssrf_guard.validate_url(allow_private=True)` — LAN targets legit (admin-approved), loopback/metadata blocked (else the envelope could hit the cortex API bypassing the allowlist). Redirects not followed (allowlist escape). `APP_HTTP_TIMEOUT`, 20 MB response cap. **Enforcement lives in `app_task_service.execute_app_http`** — one shared path for this endpoint AND http task steps. Reference consumer: `coding/cortex-paperless`.
- **`config` read (implicit, all apps)** — `GET /apps/{id}/api/platform/config` returns NON-secret config values (`public_config()`); secret-typed values never cross this boundary (nor do they enter task context — only non-secret config is interpolatable).
- **`storage`** — per-app SQLite KV (`{apps_dir}/{app_id}/storage.sqlite`, `app_storage_service.py`). JSON values, path-like keys (`_KEY_RE`), quotas `APP_STORAGE_MAX_MB`/`APP_STORAGE_MAX_VALUE_KB`. Routes: `GET/PUT/DELETE /apps/{id}/api/platform/storage/{key}` + prefix list `GET …/storage?prefix=&limit=&after=`. `missing(keys)` is the bulk dedup primitive used by task fan-out. No cross-app access by construction; dies with uninstall.
- **`tasks`** — declarative step-queues executed server-side (`app_task_dsl.py` = pure validation/interpolation/conditions/chunk policies; `app_task_service.py` = engine). Definition: `{name, schedule?: {everyMinutes}, concurrency, setup?: [steps], items: [{vars}]|{from|fromEach, vars, limit?, skipIfStored?}, steps?: [steps], finally?: [steps]}` — `fromEach: ["$f0.items", …]` concatenates several listings into one fan-out pool (one webdav step per synced folder; a `when`-skipped listing resolves null and is ignored). Note `skipIfStored` dedups against STORAGE only — duplicates within one run's pool are not cross-deduped (fine when each file appears in exactly one listing). Step vocabulary: `http` (shared gates + `paginate: {items, next, maxPages, keyBy}` (output includes `last` = final page body for delta cursors; dotted keys like `@odata.nextLink` match literally) + optional `auth: {bearer|basic: ref}` — a run-minted credential, e.g. an OAuth refresh response, that overrides config-injected auth for that request; config secrets are untemplatable so it can't leak them), `webdav` ({url, depth?: 0|1|"infinity", filter?: "files"|"dirs", auth?} — PROPFIND through the same http gates, multistatus XML parsed server-side to `{items: [{href, name, etag, lastModified, size, contentType, isDir, fileId}], count}` (fileId = oc:fileid or deterministic href hash), self entry dropped; the DSL has no XML vocabulary otherwise — built for cloud-folder sync apps), `cortex` ({method, path, body?|multipart?} — endpoint_allowed + minted key via loopback upstream; multipart from a text ref OR `multipart: {fromUrl, method?: GET|POST, headers?, auth?, followRedirect?, filename, contentType?}` binary passthrough (followRedirect = ONE hop via `follow_upload_redirect`: https+public-only target, SSRF allow_private=False, NO headers forwarded — for Graph-style 302s to variable pre-authenticated hosts; safe only here because bytes stream into the upload, never into app-readable context) — fetches via execute_app_http and streams the bytes straight into the upload without entering step context; requires http capability; contentType defaults to the upstream response's), `llm` (prompt templates; `chunk.words` + `validate` length-ratio/word-overlap, retry-once-else-keepOriginal — yt-transcriber's rules as platform policies; metered via llm_config factory, capped `APP_TASK_LLM_CALLS_PER_RUN`), `store` (get/put/delete/list), `template` (text or conditional lines), `skipItem` (item steps only). Refs `$setup.<id>.path` / templates `{path|slug|ext|default:x|join|pluck:f|…}` (`ext` = lowercase file extension, for type-filtering listings without MIME types, e.g. Dropbox); conditions `{empty|notEmpty|eq|contains|and|or|not…}`. Engine: setup once → fan-out (bulk `skipIfStored` dedup) → item worker pool (task `concurrency`, global `APP_TASKS_GLOBAL_CONCURRENCY` semaphore, per-item error isolation) → `finally` (cursor writes; skipped on pause/cancel). Persistence: one JSON per task at `{apps_dir}/{app_id}/tasks/{task_id}.json` (throttled writes); boot resume via `resume_interrupted()`; per-minute `scheduler_loop()` re-runs due scheduled tasks (headless cron — the paperless scheduled sync). Control: `PATCH …/tasks/{id}` `{action: pause|resume|cancel|retryFailed|runNow}`; admin mirrors under `/api/admin/apps/{id}/tasks*`. Step outputs capped (`APP_TASK_STEP_OUTPUT_MAX_KB`) — big artifacts go to storage. If the DSL can't express it → the app is Class C (ECOSYSTEM.md §5.2).
- **`llm`** — gates llm task steps (no standalone endpoint yet); runs the EXTRACTION tier (`get_extraction_llm_config()`, falls back to main config) — task steps are bulk mechanical work, and the chat model is often a slow reasoning model — via the `make_async_openai_client` factory → unit-metered + Langfuse-traced automatically. NEVER close the factory's client (it's cached backend-wide; closing poisons ask too — live-found regression, tested).

## Registry consumer

`app_registry_service.py`: fetches the catalog from `APP_REGISTRY_URL` (an aggregated `index.json` in mocaOS/cortex-registry; 5-min cache, `?refresh=true` busts), filters `status: active`. Install path (`POST /api/admin/apps/registry/install {slug}`): streams the release artifact with the LISTED size as the hard cap (a lying Content-Length can't over-buffer), verifies byte length + **sha256 against the catalog's pin before `install_from_zip` sees the bytes** — checksum mismatch = 502, nothing unpacked. `GET /api/admin/apps/registry` joins listings with local install state (installed_version/update_available). Frontend: "Browse Registry" section in AppsManager (lazy fetch, scope/endpoint/capability chips, Install/Update buttons, reuses the config-wizard-on-needs_setup flow). Registry side (schemas, CI that re-verifies artifacts, submission flow, and the browse site + JSON API under `site/`): `cortex-registry/README.md`. Tests: `backend/tests/test_app_registry.py` (verified install, checksum-mismatch refusal, size-lie refusal, cache/refresh, gates). `tests/test_manifest_conformance.py` runs the cross-validator corpus from `cortex-registry/conformance/` — validate_manifest, the template's validate.mjs, and the registry validator all enforce the same floor; **changing a manifest rule means updating all three + a corpus case**. Admin task oversight UI: `AppTasksPanel.tsx` inside AppsManager's expanded details (status/schedule/counts + pause/resume/cancel/retryFailed/runNow/delete; polls while active; hidden when an app has no tasks).

## Security model (the part not to regress)

1. **No real key ever reaches the browser.** Install mints a dedicated scoped API key (`app:{id}`, created_by `"apps"`; READ or READ+MANAGE from `keyScope`; collection-restricted from `collections`). The plaintext is retained server-side in `install.json` — Fernet-encrypted when `ENCRYPTION_KEY` is set — because the proxy must present it upstream. Delete revokes the key.
2. **App tokens** (`cat_` prefix): short-lived (`APP_TOKEN_TTL_SECONDS`, default 900s) HMAC tokens, payload `{v, app, principal, role, iat, exp, jti}`. Signing secret derived from `SESSION_SECRET` (fallback `ENCRYPTION_KEY`/`ADMIN_API_KEY`). **The prefix is bound into the HMAC** (`_sign` signs `prefix+body`) so a grant token (`cag_`) can't be re-labeled `cat_` and replayed as an app token (that would skip revocation — token type confusion). `validate_token` also requires a recognized principal: `owner` (logged-in launcher), `link:{grant_id}` (share links); `appuser:{id}` reserved. Tokens validate ONLY at the app proxy — never on main API routes. The owner-token endpoint requires the caller to hold MANAGE when the app's `keyScope` is `read_write` (else a READ key could borrow the app's write-scoped minted key).
3. **Sandboxed iframe**: launcher and share shell embed apps with `sandbox="allow-scripts allow-forms allow-downloads"` and **never** `allow-same-origin` → opaque origin, no cookie access. Static serving is cookie-less by design. Token handshake via postMessage: app sends `cortex:ready`/`cortex:token:renew`, host replies `{type:"cortex:token", token}`.
4. **Proxy allowlist**: `/apps/{id}/api/cortex/{path}` forwards to `APP_PROXY_UPSTREAM` (self-loopback) with `X-API-Key` attached, but only for paths prefix-matching `cortex.endpoints` (declared `ask` covers `ask/stream`); everything else 403. `endpoint_allowed` **fully URL-decodes and rejects `..`/`.`/empty segments** before matching — otherwise an encoded `documents/..%2fsearch` passes the prefix check and the upstream normalizes it to `search`, escaping the allowlist (fixed defect). Forwarding via httpx with `read=None` timeout and raw chunk relay — **SSE streams pass through unbuffered** (chat apps break otherwise).
5. **Per-app CSP** on every served file: `default-src 'self'`, `connect-src 'self'` + resolved `externalHosts`. Zip safety at install: path traversal rejected, compressed size capped (`APP_MAX_PACKAGE_MB`), uncompressed capped at 4x.
6. **Share-link grants** (`cag_` prefix): signed, revocable records in `install.json`, optional expiry, role viewer/editor. `/a/{app_id}?g={grant}` serves a minimal cookie-less shell that exchanges the grant for an app token (`POST /api/apps/{id}/grant-exchange`, unauthenticated) and brokers renewals. Revoking a grant also kills already-issued tokens from it (checked at validate time). Visitors hold nothing that validates anywhere but the app proxy — they cannot reach the cortex UI or main API.

## Backend surface

`services/app_service.py` (`get_app_service()` singleton): lifecycle (`install_from_zip` — upgrades keep key/config/grants; delete stops live task runs then revokes the key), config (mask-preserving saves like skills), tokens/grants, `resolve_static` (SPA fallback to entry, traversal-safe), `csp_header`, `endpoint_allowed`, `upstream_api_key`. Raises `AppValidationError` carrying ALL issues (fail-fast philosophy, surfaced as 400 `{detail:{issues}}`). Companions: `app_storage_service.py` (KV), `app_task_dsl.py` + `app_task_service.py` (tasks; `TaskDefinitionError` also surfaces as 400 `{detail:{issues}}`).

Endpoints in `main.py` (Apps section, before web-crawl): admin CRUD (`/api/admin/apps*`, install is multipart) + admin task oversight (`/api/admin/apps/{id}/tasks*`), `POST /api/apps/{id}/token` (owner, `require_read_permission`), `POST /api/apps/{id}/grant-exchange` (public), proxy `api_route /apps/{id}/api/cortex/{path}` and the platform routes (config/http/storage/tasks — all registered BEFORE the static catch-all `GET /apps/{id}/{path}`), `GET /a/{id}` share shell. `_require_apps_enabled()` gates all of them. Lifespan startup (when `ENABLE_APPS`): `resume_interrupted()` + `scheduler_loop()` background task.

## Frontend surface

- `components/admin/AppsManager.tsx` (+ `AppConfigModal`, `AppGrantsModal`) on the Settings page — hides entirely when `GET /api/admin/apps` 404s (flag off).
- Launcher grid at `/apps`; iframe host at `/apps/launch/{appId}` (owner token handshake). Routing split: the Next page `/apps` and `/apps/launch/*` are filesystem routes; `afterFiles` rewrites send `/apps/{appId}/*` and `/a/*` to the backend.

## Tests

`backend/tests/test_apps.py` (38 tests): manifest validation, install/upgrade/delete lifecycle with fake key service, zip safety, token roundtrip/expiry/tamper, grant exchange + revocation-kills-tokens, allowlist prefix semantics, static traversal, CSP config resolution, secret masking, platform capabilities (http host resolution, auth-header injection, config secret exclusion, endpoint gating), HTTP endpoint gates (404-when-disabled, 401/403 proxy, share shell). **Security regressions** (from the 2026-07-20 review): allowlist traversal-bypass rejection, share-shell manifest XSS escaping, grant→app token type-confusion, owner-token MANAGE gate, grant-exchange rate limit. Upstream proxy forwarding is exercised live via the verify skill.

`backend/tests/test_app_tasks.py`: storage KV/quotas, DSL validation matrix, interpolation/refs/conditions/chunk policies, and engine end-to-end runs against fakes — including the two ECOSYSTEM §5.2 acceptance shapes (paperless-style scheduled sync: paginate → dedup → conditional skip → template → cortex multipart → cursor write; yt-transcriber-style batch: transcribe → chunked llm refine with keepOriginal policy → store), plus cortex-step allowlist rejection, llm-call cap, output-size cap, lifecycle actions, scheduler due-logic, boot resume, endpoint capability/role gates. **Second security review (tasks capability, 2026-07-20): no P0/P1;** fixed + regression-tested: pagination byte-budget (accumulation bounded DURING the loop, maxPages ≤ 50 — was buffering up to maxPages × 20 MB headless), setup/finally steps now hold the global concurrency slot, http-step `headers` forwarded with a denylist (`Authorization`/`Cookie`/`Host`/framing dropped; config-injected auth always wins — was validated-but-silently-ignored), storage `_db_path` re-validates the app-id slug (defense-in-depth).
